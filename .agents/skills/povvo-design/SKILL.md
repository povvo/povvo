# Povvo Spec-Scan Router

This skill applies the Povvo Spec-Scan design language. The package root is the active skill folder: `SKILL.md` sits beside `DESIGN.md`, and slash-invoked users should get the language directly.

## Procedure

1. Read `DESIGN.md` first before creating, reviewing, or repairing any output.
2. Load only the fixed package file needed for the current task.
3. Apply foundations, tokens, components, patterns, accessibility, applications, and output rules for creation.
4. Use proofs, sources, and handoff only for validation, disputed drift, provenance, readiness, or risk questions.
5. Reject invented components, raw values where semantic tokens exist, unsupported states, saturated cyan field drift, people or sport subject matter, and prompt details not allowed by `DESIGN.md`.

## Support Files

- `DESIGN.md`: primary source-free reconstruction contract.
- `sources/`: source inventory and source-mode handling for extraction validation.
- `foundations/`: visual DNA, seven-domain extraction, imagery, voice, and brand expression.
- `tokens/`: primitive, global, alias, semantic, component, mode, platform, transform, and drift rules.
- `components/`: reusable component-family contracts.
- `patterns/`: behavioral, perceptual, language, domain, and platform patterns.
- `accessibility/`: contrast, CVD, keyboard, focus, and motion obligations.
- `applications/`: composition rules by use case.
- `outputs/`: documentation, AI-readiness, and prompt contracts.
- `proofs/`: evidence map, ontology coverage, proof report, and validation record for validation only.
- `handoff/`: readiness, provenance, residual risks, and next actions.

## Guardrails

- `DESIGN.md` is the reconstruction contract. Proof files validate traceability and gaps; they are not creation guidance.
- Do not reopen original sources unless the user asks for a new extraction pass.
- Do not copy source assets into production output unless rights and custody allow it.
- Do not invent components, raw values, unsupported states, or prompt details outside `DESIGN.md`.
- Preserve accessibility obligations and mark gaps instead of guessing.
- Keep the interrupted-aperture `P` exact, cyan subordinate, and source subjects excluded. Permit no substitute primary mark.

## Output Contract

Return the created or reviewed artifact, the `DESIGN.md` rules applied, support files consulted, validation or inspection performed, unresolved gaps, and residual risks. If generating images, quote exact text, state aspect ratio, and report failure checks for extra text, palette, subject, crop, and accessibility.

## Constraints Reminder

Before responding, verify:

1. `DESIGN.md` was read first and treated as the primary design-language contract.
2. Support files were loaded only when needed for the task.
3. Components, tokens, patterns, prompts, and accessibility rules were not invented outside the package.
4. Proof, provenance, and rights limits stayed in validation/support files; production guidance stayed source-free.
5. Validation evidence is reported only when making readiness or proof claims.
6. No people, sport subject matter, product or lifestyle imagery, saturated cyan fields, generic dark UI, alternate logo geometry, or substitute symbol entered the output.
