# Povvo Spec-Scan

## Mandatory Reading Order

This file is the canonical entry point for the Povvo design language. Read the following files in order before creating, reviewing, repairing, or generating a Povvo artifact:

1. [`design/01-core-contract.md`](design/01-core-contract.md)
2. [`design/02-foundations.md`](design/02-foundations.md)
3. [`design/03-interaction-components.md`](design/03-interaction-components.md)
4. [`design/04-output-governance.md`](design/04-output-governance.md)

The split is an agent-context optimisation only. Together, the four files are the complete canonical `DESIGN.md` contract from the Povvo source package. Support files under `foundations/`, `components/`, `patterns/`, `tokens/`, `accessibility/`, `applications/`, `outputs/`, `handoff/`, `proofs/`, and `sources/` provide deeper implementation and evidence.

## Non-Negotiable Identity Contract

- The language name is exactly **Povvo Spec-Scan**.
- The interrupted-aperture `P` is the only primary mark. Its geometry is identical inside `POVVO` and when used alone.
- The canonical set is filled `POVVO`, outline-construction `POVVO`, filled standalone `P`, and outline standalone `P`.
- Black and off-white dominate. Muted cyan is an instrument for line, focus, scan, construction, or rare accent.
- Build from named tokens, component recipes, and pattern rules before inventing new choices.
- Never rely on colour, motion, texture, position, or tiny text as the only carrier of required information.
- People, sport subject matter, alternate logo geometry, unrelated icons, modular blobs, saturated cyan brand fields, glossy control-room styling, and product-object stand-ins are forbidden.

## Source-Free Use

This skill is intentionally source-free at runtime. The design contract and support files are sufficient to reproduce, assess, and govern Povvo work without opening proof, provenance, or original-source material. The two images under `showcase/assets/` are compact visual anchors, not substitutes for the written contract.
